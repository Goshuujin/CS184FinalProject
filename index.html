Final Project: Solar System Simulation

David Allen

I plan to create a fully animated simulation of the solar system. The simpulation will include texture mapped planets as well as a camera fly by of the planets while in simulation.

Background

The problem I intend to solve is twofold. First I hope to learn about animation and efficiently interating with the scene and the camera. Secondly I want to simulate an actual physics modeled environment that could be used not only for scientific predictions, but for fun as well. This is important to me because I have always wanted to create a physics engine that was not only accurate but plesant to interact with. I also enjoy video games and would like to learn as much as I can to further my goal of building a 3D game. Others should care bout this because space exploration would not be possible without accurate simulations.

Resources

I plan to build this from scratch using C++ and OpenGL. I have yet to find many resources for making this possible, however I know I will most likely be leaning on the OpenGL documantation pretty heavily. I will also be using textures from the NASA/JPL solar system simulator. I will also be using my MacBook Pro or the lab computers for the hardware.

Goals and Deliverables

What I plan on delivering is a demo of the solar system simulation using an animated camera fly through, starting from Pluto and going all the way into the Sun. I order to accomplish this I must render the planets using mipmapping, create a Sun that is esentially a spherical light source to illuminate the scene, calculate and update the scene for animation at every time step, and create a camera fly by routine that takes into account its distance from a target and target location. What I hope to deliver as well is more interactivity with the scene by letting the user control the time step of the simulation and creating a initial over head view of the solar system with labels showing the location of the planets in real time. In order to accomplish these goals I will need to create an animated 2D overlay when the camera is at a certain position along with event handlers that move the camera to that position (as well as some to control the speed of the simulation). If I acomplish all of these goals I hope to add a few bells and whistles such as putting the rings on Saturn and Uranus as well as the atmosphere around Earth.

Schedule

  • Render the solar system with texture mapped planets.
  • Accurately animate and simulate the solar system.
  • Create a camera fly by of the simulation.
  • Create an birds eye view with 2D overlay labeling planets.
  • Speed up or slow down the simulation with user input.
